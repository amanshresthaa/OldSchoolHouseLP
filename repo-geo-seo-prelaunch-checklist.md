# Repo GEO + SEO Pre-Launch QA Checklist

Use this checklist before launching the unpublished repo version of The Old School House site.

---

## 1. Core Strategic Positioning

- [ ] Homepage clearly leads with `traditional pub + Nepalese kitchen + Stony Stratford`
- [ ] New-management / Sanjog positioning is approved and consistent everywhere it appears
- [ ] The site competes as a differentiated local venue, not a generic pub
- [ ] Primary conversion goals are clear: bookings, calls, directions, private-hire enquiries

---

## 2. Trust, Proof, and Accuracy

- [ ] Remove any unverified testimonial-style quotes
- [ ] Replace proof blocks with real Google reviews, attributed quotes, or no quotes at all
- [ ] Address, phone, email, map link, and booking link are all verified
- [ ] Opening hours are confirmed before launch, or intentionally held back everywhere
- [ ] Dog-friendly, family-friendly, accessibility, and event claims are all operationally confirmed
- [ ] Private-hire, wakes, and buffet details are current and approved

---

## 3. Technical SEO Foundations

- [ ] `robots.txt` resolves correctly and points to `sitemap.xml`
- [ ] `sitemap.xml` only includes routes intended for indexing
- [ ] Canonical URLs are correct on all published pages
- [ ] Redirect-only pages remain non-indexable and behave correctly
- [ ] Metadata titles and descriptions are unique across core pages
- [ ] Open Graph and Twitter metadata are consistent with final positioning
- [ ] No legacy live-site copy remains in metadata or structured data

---

## 4. Structured Data and GEO Readiness

- [ ] Homepage `BarOrPub` schema is valid
- [ ] Menu page `Menu`, `MenuSection`, and `MenuItem` schema is valid
- [ ] FAQ schema is added to all FAQ-heavy pages
- [ ] Breadcrumb schema is added sitewide if planned
- [ ] Event schema is only used for real, current events
- [ ] `sameAs` links point to real active profiles
- [ ] `llms.txt` is added
- [ ] `llms-full.txt` is added if referenced
- [ ] Key pages contain short answer-style blocks that are easy for AI systems to quote

---

## 5. Route-by-Route QA

### `/`

- [ ] Hero framing is final and approved
- [ ] Homepage proof bar uses verified claims only
- [ ] Internal links to `/about`, `/menu`, `/nepalese-kitchen`, `/book`, `/private-hire` are present
- [ ] Any review section uses only real proof

### `/about`

- [ ] Sanjog story is approved and factually accurate
- [ ] Lapen Inns mention is approved and supported
- [ ] Page explains why the venue is different, not just its backstory
- [ ] Internal links to menu, kitchen, booking, and find-us pages are present

### `/menu`

- [ ] Live HTML menu matches the latest approved menu
- [ ] Prices are correct
- [ ] Dietary labels are correct
- [ ] PDF download works if offered
- [ ] “Start here” guidance exists for guests new to Nepalese food

### `/book`

- [ ] Booking engine link works
- [ ] Phone and email fallback are correct
- [ ] Booking reassurance copy is accurate
- [ ] High-intent routes to menu, private hire, and find-us are visible

### `/find-us`

- [ ] Address matches GBP/directory formatting
- [ ] Directions link works
- [ ] Parking wording is accurate
- [ ] Accessibility wording is accurate
- [ ] Opening hours display is correct

### `/nepalese-kitchen`

- [ ] This page clearly explains the differentiator
- [ ] Signature dish references are accurate
- [ ] Sanjog / kitchen narrative feels credible and specific
- [ ] Links to `/menu`, `/about`, and `/book` are prominent

### `/sunday-roast`

- [ ] Offer details and prices are current
- [ ] Sunday timing is correct
- [ ] Any promo wording matches actual operations
- [ ] A strong booking path is present

### `/private-hire`

- [ ] Occasion types listed are realistic
- [ ] Enquiry route works
- [ ] Capacity / setup wording is approved
- [ ] Links to wakes and group-dining pages are present

### `/events`

- [ ] “What’s on” wording matches reality
- [ ] No fake recurring schedule is implied
- [ ] Links to live sport and private hire are present

### `/live-sport`

- [ ] Sport coverage claims are accurate
- [ ] Pub-first positioning is preserved
- [ ] Booking CTA works for event-led visits

### Supporting pages

- [ ] `/dog-friendly-pub-stony-stratford` is only live if policy is confirmed
- [ ] `/family-friendly-pub-stony-stratford` is only live if policy is confirmed
- [ ] `/beer-garden-stony-stratford` reflects real outdoor offer
- [ ] `/group-dining-celebrations` is clearly distinct from private hire
- [ ] `/wakes-life-celebrations` is fully approved and up to date
- [ ] `/takeaway-menu`, `/menu-information`, and `/accessibility` are accurate and useful

---

## 6. Internal Linking

- [ ] Homepage links into all key commercial clusters
- [ ] Menu links into kitchen, Sunday roast, takeaway, and group dining
- [ ] About links into menu, kitchen, booking, and find-us
- [ ] Private-hire links into wakes and group dining
- [ ] Find-us links into booking and high-intent occasion pages
- [ ] Footer emphasizes core commercial routes, not only support/legal routes

---

## 7. External Entity Alignment

- [ ] Google Business Profile matches the launched site messaging
- [ ] TripAdvisor information is aligned
- [ ] CAMRA / Visit Stony / FANZO / local directory citations are aligned where editable
- [ ] Social profiles reflect the same positioning as the launched site
- [ ] External references do not materially contradict the new-management story

---

## 8. Content Quality Checks

- [ ] No thin doorway-style pages are being published
- [ ] Redirected awareness pages stay redirected unless expanded properly
- [ ] Copy is specific, concrete, and venue-grounded
- [ ] No placeholder wording remains
- [ ] No claims rely on fabricated authority

---

## 9. Design and UX Checks

- [ ] Mobile navigation works cleanly
- [ ] Sticky booking CTA works
- [ ] Maps and consent handling behave correctly
- [ ] Buttons are consistent across the site
- [ ] CTAs are visible above the fold on major pages
- [ ] Images are real, compressed, and relevant if included

---

## 10. Final Launch Gate

- [ ] Content approved
- [ ] Operational details approved
- [ ] Structured data checked
- [ ] Internal linking checked
- [ ] External citations reviewed
- [ ] Validators passed
- [ ] Launch-ready decision recorded
